## Design Context

### Users
싱글 유저 — 본인 자산을 직접 관리하는 개인. 주로 데스크탑에서 저녁/주말에 사용. 연말 결산이 핵심 목적이며, 일상적으로는 포트폴리오 현황 확인과 거래 내역 기록에 사용한다. 금융 데이터에 익숙하지만 전문 트레이더는 아님.

### Brand Personality
**개성 있되 절제된, 차분하고 명확한, 나만을 위한**

- 카지노/잭팟 테마는 유지하되 지배하지 않는다 — 포인트로서의 개성, 앱의 기반은 아님
- 복잡한 금융 데이터를 단순하고 명확하게 전달하는 것이 최우선
- "내가 내 자산을 장악하고 있다"는 자신감, "복잡한 걸 단순하게 봤다"는 만족감

### Aesthetic Direction
- **테마**: 라이트 기반 (절제된 빛, 깨끗한 배경)
- **방향**: Editorial/refined — 금융 앱의 구조적 명확함 + 잭팟 브랜드의 포인트적 개성
- **안티레퍼런스**: AI-generated 느낌 (gradient text, glassmorphism, border-left stripe, hero metric layout, generic card grids, neon accents on dark)
- **참고 방향**: 전문성이 느껴지는 개인 도구 — 블룸버그처럼 딱딱하지 않고, 뱅킹 앱처럼 무미건조하지 않으며, AI 데모처럼 과시적이지 않은

### Design Principles
1. **명확함 우선** — 차트, 수치, 레이아웃 모두 "한눈에 파악 가능"해야 한다. 장식이 데이터를 방해하면 제거한다.
2. **개성은 포인트로** — JACKPOT 브랜딩, 슬롯머신, 시머 효과는 살리되, 핵심 UI(카드, 테이블, 폼)는 간결하고 기능적으로.
3. **AI 슬롭 제거** — border-left stripe, gradient text, glassmorphism, generic hero metric, identical card grids는 모두 제거 대상.
4. **공간으로 위계를** — 동일한 패딩, 동일한 카드 반복 지양. 중요한 정보는 더 많은 공간으로 강조.
5. **데이터 시각화의 충실함** — 차트는 정보를 전달하기 위해 존재한다. 장식적 스파크라인, 의미 없는 그라디언트는 금지.
