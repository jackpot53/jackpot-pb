## Spacing Scale

Declared values (multiples of 4 only). All values map to Tailwind utility classes.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | gap-1 / p-1 | Icon gaps, inline badge padding |
| sm | 8px | gap-2 / p-2 | Compact element spacing, table cell padding |
| md | 16px | gap-4 / p-4 | Default card padding, form field spacing |
| lg | 24px | gap-6 / p-6 | Section padding (matches existing `main p-6`) |
| xl | 32px | gap-8 / p-8 | Dashboard section breaks |
| 2xl | 48px | gap-12 | Major layout gaps between dashboard rows |
| 3xl | 64px | gap-16 | Page-level spacing (reserved, not currently used) |

Exceptions:
- Stat card inner padding: 24px (`p-6`) — matches shadcn Card default
- Table row height: minimum 44px to meet touch-target accessibility standard

## Typography

All sizes use Inter via `var(--font-sans)`. Weights are limited to exactly 2.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Table cell content, labels, secondary text |
| Label | 16px (text-base) | 400 (normal) | 1.5 | Form labels, nav items, card body copy |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Card titles, section headings |
| Display | 28px (text-3xl) | 600 (semibold) | 1.1 | Stat card primary values (total asset KRW) |

Note: The existing sidebar uses `text-sm` for nav items and `text-xs font-semibold` for the category label — Phase 3 does not modify nav typography.

## Color

All values reference existing CSS custom properties from `app/globals.css` (oklch tokens, shadcn base-nova neutral palette).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` (oklch 1 0 0 / white) | Page background, main content area |
| Secondary (30%) | `bg-card` / `bg-sidebar` (oklch 1 0 0 / oklch 0.985 0 0) | Stat cards, pie chart section card, sidebar |
| Accent (10%) | `bg-primary` (oklch 0.205 0 0 / near-black) | Active nav item, primary CTA button only |
| Destructive | `var(--destructive)` (oklch 0.577 0.245 27.325 / red) | No destructive actions in Phase 3 — reserved token, not rendered |

Accent reserved for: active sidebar nav item background only. NOT used on stat cards, table rows, or chart elements.

### Semantic color overrides (Phase 3 specific)

These use Tailwind color utilities, NOT the shadcn token system, because financial gain/loss semantics require color meaning beyond the neutral palette:

| Semantic | Color class | Usage |
|----------|-------------|-------|
| Positive return | `text-emerald-600` (light) / `text-emerald-400` (dark) | Return % when value > 0, gain/loss KRW when positive |
| Negative return | `text-red-600` (light) / `text-red-400` (dark) | Return % when value < 0, gain/loss KRW when negative |
| Neutral / zero | `text-muted-foreground` | Return % when exactly 0%, MANUAL assets with no price |
| Stale price indicator | `text-amber-600` bg-`amber-50` border-`amber-200` | "오래된 가격" badge when priceCache is stale |

### Pie Chart Color Palette (recharts)

Asset-type slices use the existing chart tokens from globals.css, extended with semantic color for recognizability:

| Asset type | Color | Hex approx |
|------------|-------|-----------|
| stock_kr / stock_us | `#3B82F6` (blue-500) | Matches existing badge pattern |
| etf_kr / etf_us | `#6366F1` (indigo-500) | Matches existing badge pattern |
| crypto | `#F97316` (orange-500) | Matches existing badge pattern |
| real_estate | `#22C55E` (green-500) | Matches existing badge pattern |
| savings | `#6B7280` (gray-500) | Matches existing badge pattern |

Pie chart uses `var(--chart-1)` through `var(--chart-5)` as fallback tokens if asset type colors are not available.

## Component Specification

### Stat Card (`dashboard-stat-card.tsx`)

```
Card (p-6)
  CardHeader (pb-2)
    label: 14px / normal / muted-foreground   e.g. "총 자산 (KRW)"
  CardContent
    primary value: 28px / semibold / foreground   e.g. "₩124,500,000"
    secondary line: 14px / normal / muted-foreground  e.g. "+₩12,340,000 (10.9%)"
      → color: text-emerald-600 if positive, text-red-600 if negative
```

Loading state: replace primary value with `Skeleton` (h-8 w-32 rounded-md).
Stale state: no special treatment on stat cards — stale indicator appears per-row in performance table only.

### Allocation Pie Chart Card

```
Card (p-6)
  CardHeader: "자산 배분" (Heading, 20px semibold)
  CardContent
    PieChart (recharts) width=300 height=300
      innerRadius=60 outerRadius=100 (donut style)
      Tooltip: show asset type label + KRW value + %
      Legend: below chart, horizontal, asset type labels
  No data state: centered text "보유 자산이 없습니다"
```

### Asset Breakdown List (right of pie chart)

```
Card (p-6)
  CardHeader: "유형별 합계" (Heading, 20px semibold)
  CardContent
    list of rows per asset type:
      AssetTypeBadge | KRW total (16px semibold) | % of portfolio (14px muted)
    separator between rows
  No data state: muted text "데이터 없음"
```

### Performance Table (`performance-table.tsx`)

```
Card (p-6)
  CardHeader
    "종목별 성과" (Heading, 20px semibold)
    sort state indicator: "수익률 순" (14px muted, right-aligned)
  CardContent
    Table
      TableHead row (14px semibold):
        종목명 | 유형 | 수량 | 평단가(KRW) | 현재가 | 평가금액(KRW) | 수익률(%)
        → Clickable headers: 평단가, 현재가, 평가금액, 수익률
        → Sort icon: ChevronUp/ChevronDown (lucide, 14px) on active column
      TableBody rows (14px normal):
        - 종목명: asset name, 16px normal foreground
        - 유형: <AssetTypeBadge>
        - 수량: right-aligned, monospace
        - 평단가: right-aligned KRW formatted integer
        - 현재가: right-aligned KRW formatted integer
          → MANUAL assets: latest ManualValuation value, no refresh indicator
          → LIVE stale: value + <StalePriceBadge cachedAt={timestamp}>
        - 평가금액(KRW): right-aligned, 16px semibold
        - 수익률(%): right-aligned, text-emerald-600 if positive / text-red-600 if negative
    Default sort: 수익률 내림차순 (D-13)
```

Empty state (no holdings): centered in CardContent, 48px top padding:
  - Heading: "보유 종목이 없습니다" (16px semibold, foreground)
  - Body: "자산 메뉴에서 종목을 추가하고 거래 내역을 입력하세요." (14px muted)
  - CTA button: variant="outline" → label "자산 추가" → href="/assets/new"

### Stale Price Badge (`stale-price-badge.tsx`)

```
inline element, rendered inside table cell next to current price value:
  Badge variant="outline" className="bg-amber-50 border-amber-200 text-amber-600 text-xs ml-2"
  content: "오래된 가격 · {relative time}"
  e.g. "오래된 가격 · 3시간 전"
```

Tooltip (shadcn Tooltip) on hover: "마지막 업데이트: {cachedAt formatted as 'YYYY-MM-DD HH:mm'}"

## Loading States

Price fetch is on-demand (D-01). Loading occurs on dashboard initial render before API response returns.

| Element | Loading treatment |
|---------|-------------------|
| All 4 stat cards | Replace each primary value with `<Skeleton className="h-8 w-32" />` |
| Pie chart | Replace with `<Skeleton className="h-[300px] w-full rounded-xl" />` |
| Breakdown list | 3× skeleton rows: `<Skeleton className="h-5 w-full" />` |
| Performance table | 5× skeleton rows, each column cell has `<Skeleton className="h-4 w-full" />` |
| Exchange rate | Stat card "총 자산 (USD)" shows skeleton until FX rate loads |

Loading skeleton uses shadcn `Skeleton` component (pulse animation via shadcn base-nova).

## Number Formatting Contract

Financial values are stored as BIGINT integers and must be formatted consistently (source: Phase 1 D-04).

| Value type | Storage | Display format | Helper function |
|------------|---------|----------------|-----------------|
| KRW amount | integer won | `₩N,NNN,NNN` (Intl.NumberFormat ko-KR, 0 decimals) | `formatKrw(bigint)` |
| USD amount | integer cents (÷100) | `$N,NNN.NN` (Intl.NumberFormat en-US, 2 decimals) | `formatUsd(bigint)` |
| Quantity | integer ×10^8 (÷1e8) | up to 8 decimal places, trailing zeros stripped | `formatQty(bigint)` |
| Return % | computed float | `+N.N%` / `-N.N%` / `0.0%` | `formatReturn(number)` |
| Exchange rate | integer ×10000 (÷10000) | internal only, not displayed raw | `formatFxRate(bigint)` |
