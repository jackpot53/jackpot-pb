## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Compact element spacing (nav item padding, card inner) |
| md | 16px | Default element spacing (form fields, card padding) |
| lg | 24px | Section padding (card headers, page section gaps) |
| xl | 32px | Layout gaps (between dashboard sections) |
| 2xl | 48px | Major section breaks (page-level top padding) |
| 3xl | 64px | Page-level spacing (not used this phase) |

Exceptions:
- Touch targets (table rows, nav links): min-height 44px — matches existing `min-h-[44px]` on TableRow (source: `performance-table.tsx`)
- Progress bar height: 8px (2× xs) — intentionally sub-scale for inline display

## Typography

Established pattern from existing components (`dashboard-stat-card.tsx`, `performance-table.tsx`, `annual-return-chart.tsx`).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (text-base) | 400 (font-normal) | 1.5 | Table cell content, dialog body text |
| Label | 14px (text-sm) | 400 (font-normal) | 1.5 | Muted secondary info, stat card labels, table headers |
| Heading | 20px (text-xl implied via CardTitle) | 600 (font-semibold) | 1.2 | Card titles, section headings, page headings |
| Display | 30px (text-3xl) | 600 (font-semibold) | 1.0 (leading-tight) | Dashboard stat card primary values |

Additional established pattern:
- Return % values: 14px, color-coded (text-emerald-600 positive / text-red-600 negative)
- Tooltip body: 14px (text-sm) with card background
- Chart axis ticks (12px recharts internal) are not a declared typography token — inherited from existing recharts component configuration; new UI elements use the 4-size scale above

Source: `dashboard-stat-card.tsx`, `performance-table.tsx`, `annual-return-chart.tsx`

## Color

Tokens from `app/globals.css` (oklch, light mode canonical values shown as role descriptions).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch 1 0 0 = white) | Page background, table rows |
| Secondary (30%) | `--card` / `--sidebar` (oklch 0.985 0 0 = off-white) | Cards, sidebar, dialogs |
| Accent (10%) | `--primary` (oklch 0.205 0 0 = near-black) | Active nav item bg, primary buttons, progress bar fill |
| Destructive | `--destructive` (oklch 0.577 0.245 27.325 = red) | Delete goal button, delete confirmation dialog |

Accent reserved for:
- Active sidebar nav item background (`bg-primary text-primary-foreground`)
- Primary action buttons ("목표 추가", "변경 저장" in goal dialog)
- Goal progress bar fill (achieved portion)

Semantic color additions (not destructive, not accent):
- Positive return: `text-emerald-600` — return % ≥ 0 in performance table, goal achievement ≥ 100%
- Negative return: `text-red-600` — return % < 0 in performance table
- Goal overachievement (≥ 100%): `text-emerald-600` on achievement % label, progress bar capped at 100% width but label shows actual %

Source: `app/globals.css`, `performance-table.tsx`, `dashboard-stat-card.tsx`

## Interaction Contracts

### Goal CRUD (D-01, D-02)

| Interaction | Behavior |
|-------------|----------|
| "목표 추가" button click | Opens Dialog with goal form (controlled open state) |
| Goal form submit (valid) | Server Action `createGoal` → `revalidatePath('/goals')` → dialog closes |
| Goal form submit (invalid) | Inline `FormMessage` errors shown; dialog stays open |
| Goal row edit icon click | Opens same Dialog pre-populated with goal data (`updateGoal` Server Action) |
| Goal delete: trigger | "삭제" button opens confirm Dialog (separate component) |
| Goal delete: confirm | `deleteGoal` Server Action → `revalidatePath('/goals')` → dialog closes |
| Goal delete: cancel | Dialog closes, no action |

### Dashboard Goals Section (D-03, D-04)

| Interaction | Behavior |
|-------------|----------|
| Section visibility | Hidden entirely when `goals.length === 0` (no empty state shown) |
| Progress bar | shadcn Progress component; value = `Math.min(achievementPct, 100)` (capped at 100 for bar width); label shows actual % including >100 |
| Achievement ≥ 100% | Achievement % label rendered in `text-emerald-600`; no other special decoration |
| Goal card click | No navigation — goals section is informational only on dashboard |

### Goals Page — Progress Chart (D-05, D-06, D-07, D-08)

| Interaction | Behavior |
|-------------|----------|
| Chart with snapshots | AreaChart (portfolio total KRW) + one `ReferenceLine y={targetAmountKrw}` per goal |
| Goal with targetDate | Additional vertical `ReferenceLine x={targetDate}` on X-axis |
| No snapshots | Static message: "아직 데이터가 없습니다. 크론 잡이 내일 첫 스냅숏을 기록합니다." |
| Tooltip hover | Custom tooltip showing date, portfolio value (formatKrw), nearest goal if within range |

### Performance Page — Filter Tabs (D-09, D-10, D-11, D-12)

| Interaction | Behavior |
|-------------|----------|
| Default tab | "전체" tab active; table shows all rows sorted by returnPct desc |
| Tab click | Filters `rows` prop to matching asset types; sort state preserved |
| Tab "주식" | Includes: stock_kr, stock_us, etf_kr, etf_us |
| Tab "코인" | Includes: crypto |
| Tab "예적금" | Includes: savings |
| Tab "부동산" | Includes: real_estate |
| Column header click | Toggles sort asc/desc for that column (existing behavior in PerformanceTable) |
| Empty filtered tab | Shows existing empty state: "보유 종목이 없습니다" + "자산 메뉴에서 종목을 추가하고 거래 내역을 입력하세요." + "자산 추가" button |

### Navigation

| Interaction | Behavior |
|-------------|----------|
| Sidebar "성과" item | Links to `/performance`; active state matches existing nav pattern (`bg-primary text-primary-foreground font-semibold`) |
| Sidebar "목표" item | Already present in `sidebar.tsx`; links to `/goals` |

## Layout Contract

### /goals Page

```
<app-shell>
  <page-header>
    <h1>목표</h1>
    <Button "목표 추가" variant="default" />
  </page-header>

  <GoalProgressChart />   ← Card, AreaChart + ReferenceLine per goal
                           (or no-snapshot message)

  <GoalList>              ← Card containing goal rows
    <GoalRow>
      name | targetAmountKrw (formatKrw) | targetDate (optional) | edit icon (aria-label="목표 수정") | delete icon (aria-label="목표 삭제")
    </GoalRow>
  </GoalList>

  <GoalDialog />          ← controlled Dialog (create or edit)
  <DeleteGoalDialog />    ← controlled confirm Dialog
</app-shell>
```

### Dashboard Goals Section (appended below existing performance table section)

```
<section>
  <h2 class="text-sm font-semibold text-muted-foreground uppercase tracking-wider mb-3">목표</h2>
  <Card>
    <CardContent>
      {goals.map(goal =>
        <div class="flex items-center gap-4 py-3 border-b last:border-0">
          <span class="flex-1 text-base">{goal.name}</span>
          <span class="text-sm text-muted-foreground">{formatKrw(goal.targetAmountKrw)}</span>
          <span class="text-sm font-semibold [emerald if ≥100%]">{achievementPct}%</span>
          <Progress value={Math.min(achievementPct, 100)} class="w-24 h-2" />
        </div>
      )}
    </CardContent>
  </Card>
</section>
```
