## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, table cell icon spacing |
| sm | 8px | Compact element spacing, form field internal gaps |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, card content padding (matches established `p-6` = 24px from Phase 1) |
| xl | 32px | Layout gaps between page sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top padding |

Exceptions: Touch targets for icon-only buttons (void/delete row actions) must be minimum 36px hit area via padding — use `p-2` (8px) around 20px icon.

Source: Default 8-point scale; `p-6` / `gap-4` patterns confirmed from Phase 1 login page.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 14px (text-sm) | 400 (regular) | 1.4 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Display | 28px (text-2xl) | 600 (semibold) | 1.2 |

Notes:
- Body (14px/400) applies to table cells, form helper text, transaction list rows, valuation history entries.
- Label (14px/400) applies to form field labels (matching Phase 1 `<FormLabel>` usage), table column headers, tab labels. Distinguish from body text using `text-muted-foreground` color — not a separate font weight.
- Heading (20px/600) applies to page-level section titles, asset name on detail page (matching Phase 1 `text-xl font-semibold` on CardTitle).
- Display (28px/600) is reserved for the asset detail page's primary value figure (current value KRW amount) and the top-level page heading if used.
- No additional font sizes. Do not introduce 12px or 18px.
- Exactly 2 font weights in use: 400 (regular) and 600 (semibold). Do not introduce 500 (medium).

Source: Phase 1 login page uses `text-xl font-semibold` for CardTitle. shadcn base-nova provides Geist Sans at these sizes.

## Color

All colors expressed as CSS custom properties from `app/globals.css`. No hardcoded hex or oklch values in component code.

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `bg-background` | Page background, main content area |
| Secondary (30%) | `bg-card` / `bg-sidebar` | Cards, sidebar, table rows (alternating via `bg-muted` for even rows) |
| Accent (10%) | `bg-primary` / `text-primary` | Reserved list below |
| Destructive | `text-destructive` / `bg-destructive` | Void transaction button, delete asset confirmation only |

Accent (`primary`) reserved for:
1. The active/selected sidebar nav item background
2. The primary CTA button label (자산 추가, 거래 추가, 가치 업데이트, 자산 저장, 거래 저장, 가치 저장) — uses shadcn `<Button variant="default">`
3. Focus ring (`ring`) on focused form inputs

Secondary interactions (edit, cancel, back) use `<Button variant="outline">` or `<Button variant="ghost">` — never accent color.

Gain/Loss semantic colors (financial context, not general UI accent):
- Gain (수익): `text-green-600` (dark mode: `text-green-400`) — only on return % and gain amount values
- Loss (손실): `text-red-600` (dark mode: `text-red-400`) — only on return % and loss amount values
- These are the only permitted uses of non-neutral non-destructive color in Phase 2.

Source: `globals.css` color system confirmed. Gain/loss semantic pattern is standard for Korean fintech UI and required by ASSET-01/D-03 (수익% column).

## Form Field Order

### 자산 추가/수정 폼

Order: 종목명 → 자산 유형 → 통화(KRW/USD) → 시세 유형(live/manual) → 티커 (live 선택 시만 표시) → 메모 (optional)

### 거래 추가/수정 폼

Order: 거래 유형(매수/매도) → 날짜 → 수량 → 단가 → 환율 (USD 자산 시만 표시, label: "거래 시 환율 (₩/＄)") → KRW 환산 미리보기 (읽기 전용, "≈ ₩{computed}") → 수수료 → 메모 (optional)

Source: CONTEXT.md D-10, D-11, D-12, D-13 — quantity decimal input, KRW storage, USD exchange rate field conditional display.

## Interaction States

### Table Row Actions

Each row in the asset list shows action icons on hover (not always visible):
- 수정 icon (`Pencil`, 16px) — opens edit form/modal — `aria-label="자산 수정"`
- 삭제 icon (`Trash2`, 16px) — triggers delete confirmation Dialog — `aria-label="자산 삭제"`

Each row in the transaction list shows:
- 수정 icon (`Pencil`, 16px) — opens edit form/modal (only for non-voided) — `aria-label="거래 수정"`
- 취소 icon (`Ban`, 16px) — triggers void confirmation Dialog (only for non-voided) — `aria-label="거래 취소 처리"`
- Voided rows display `is_voided` with strikethrough text and `opacity-50`

All icon-only action buttons must include an explicit `aria-label` attribute matching the values above. Do not rely on title attributes or tooltips as the sole accessible label.

### Loading States

- Table loading: skeleton rows (3 rows minimum) using `bg-muted animate-pulse` blocks
- Form submission: button disabled + `<Loader2 className="mr-2 h-4 w-4 animate-spin" />` inside button (matches Phase 1 pattern)
- KRW conversion preview field updates on blur of quantity/price/exchange rate fields — no debounce needed (pure computation)

### Validation

- All required fields validate on submit and on blur (using react-hook-form + zod, matching Phase 1 pattern)
- Inline error messages below each field via `<FormMessage>` (matching Phase 1 pattern)
- Quantity: stock/ETF must be whole number — show "주식/ETF 수량은 정수여야 합니다." if decimal entered
- Quantity: crypto allows up to 8 decimal places — show "소수점 8자리까지 입력 가능합니다." if exceeded
- Exchange rate: positive integer only — show "환율은 양의 정수를 입력해주세요."
